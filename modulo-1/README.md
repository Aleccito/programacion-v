# Modelos de Servicio y Despliegue en la Nube

## Objetivo

Validar la comprensión de los modelos de servicio y despliegue en la nube, diferenciándolos de la infraestructura local tradicional.

---

## Introducción

La computación en la nube permite acceder a recursos tecnológicos como servidores, almacenamiento, bases de datos y aplicaciones a través de internet, sin necesidad de mantener toda la infraestructura físicamente en la empresa.

A diferencia de la infraestructura local, donde la organización compra, instala y administra servidores propios, en la nube los recursos se consumen bajo demanda, con mayor escalabilidad, flexibilidad y reducción de costos iniciales.

---

# Modelos de Servicio en la Nube

## IaaS (Infrastructure as a Service)

Proporciona infraestructura virtualizada como servidores, redes y almacenamiento. El cliente administra el sistema operativo, aplicaciones y configuraciones.

### Ejemplos:

- Amazon EC2 (AWS)
- Microsoft Azure Virtual Machines
- Google Compute Engine

### Uso común:

Empresas que desean migrar servidores físicos a la nube manteniendo control técnico.

---

## PaaS (Platform as a Service)

Ofrece una plataforma lista para desarrollar, ejecutar y desplegar aplicaciones sin gestionar servidores.

### Ejemplos:

- Heroku
- Google App Engine
- AWS Elastic Beanstalk

### Uso común:

Equipos de desarrollo que desean enfocarse en programar sin administrar infraestructura.

---

## SaaS (Software as a Service)

Son aplicaciones completas accesibles desde navegador o internet. El proveedor administra todo.

### Ejemplos:

- Gmail
- Google Drive
- Microsoft 365
- Salesforce

### Uso común:

Empresas que necesitan herramientas listas para usar.

---

# Modelos de Despliegue en la Nube

## Nube Pública

Infraestructura compartida administrada por un proveedor externo.

### Ejemplos:

- AWS
- Microsoft Azure
- Google Cloud

### Uso común:

Sitios web, startups, aplicaciones escalables.

---

## Nube Privada

Infraestructura exclusiva para una sola organización.

### Ejemplos:

- Servidores internos virtualizados
- OpenStack privado

### Uso común:

Bancos, gobiernos, empresas con alta regulación.

---

## Nube Híbrida

Combina nube pública y privada.

### Ejemplo:

Empresa guarda datos sensibles en nube privada y usa AWS para aplicaciones web públicas.

### Uso común:

Organizaciones que buscan seguridad y escalabilidad.

---

# Comparación con Infraestructura Local

## Infraestructura Local:

- Requiere compra de hardware
- Mayor mantenimiento
- Escalabilidad limitada
- Costos iniciales altos

## Nube:

- Pago por uso
- Escalabilidad rápida
- Acceso remoto
- Menor inversión inicial

---

# Conclusión

La nube ofrece distintos modelos según necesidades técnicas y financieras. IaaS brinda control, PaaS simplifica desarrollo y SaaS entrega software listo para usar. Asimismo, las nubes pública, privada e híbrida permiten elegir entre flexibilidad, seguridad o una combinación de ambas.
